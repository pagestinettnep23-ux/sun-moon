# Base Sepolia 地址准备说明

更新日期：2026-05-14

本文档给非技术成员使用，说明进入 Base Sepolia 受控测试网预检前，需要准备哪些公开地址、每个地址做什么、哪些信息绝对不能提供。当前已完成第一次 Base Sepolia 小额广播；仍不部署主网、不接真实资金，后续广播必须单独批准。

## 1. 当前是否必须准备地址

现在不急着必须提供地址。

如果我们继续做本地 Mock、文档整理和脚本模拟，可以不提供任何钱包地址。只有当你决定进入真实 Base Sepolia 参数预检，才需要准备下面这些公开地址。

## 2. 需要准备的公开地址

| 名称 | 谁提供 | 用途 | 建议 |
| --- | --- | --- | --- |
| `HOOK_OWNER` | 你或项目方 | 管理 Hook、adapter、曲线配置的测试管理员地址 | 单独测试钱包，不要用主钱包 |
| `PROTOCOL_BUDGET_ADDRESS` | 你或项目方 | 接收 2% 协议预算费用的测试预算地址 | 单独测试钱包，不能等于 adapter |
| `DEPLOYER_ADDRESS` | 你或项目方 | 实际发送测试网部署交易的钱包公开地址 | 单独测试钱包，只放少量测试 ETH |
| `CREATE2_DEPLOYER` | 技术侧确认 | 用来预测 Hook 地址的 CREATE2 deployer 地址 | 必须固定，预测和部署必须同一个 |

其中，下一轮 CREATE2 参数预检最少需要：

```text
HOOK_OWNER=
PROTOCOL_BUDGET_ADDRESS=
CREATE2_DEPLOYER=0xE5Df76A1DF2e959C70c8b7591754a3a2a542Fb6D
```

`DEPLOYER_ADDRESS` 主要用于人工记录和广播前复核。只要还不广播，就不需要私钥。

## 3. 这些地址不能混用

建议把地址分开：

- `HOOK_OWNER`：像测试管理员钥匙，能改关键配置。
- `PROTOCOL_BUDGET_ADDRESS`：像测试收款钱包，用来接协议预算费用。
- `DEPLOYER_ADDRESS`：像施工钱包，负责发测试网部署交易。
- `CREATE2_DEPLOYER`：像固定施工工具，用错会导致预测 Hook 地址变化。

不要为了省事把所有地址都用成同一个。测试阶段可以临时简化，但广播前必须明确记录哪些地方用了同一个地址，以及为什么可以接受。

## 4. 绝对不要提供的信息

不要发给我，也不要写进文档、截图或聊天记录：

- 私钥。
- 助记词。
- keystore 文件密码。
- 完整 RPC key。
- 主网钱包地址和资产截图。
- 有真实资金的钱包操作权限。

我只需要公开地址，例如：

```text
0x1234567890abcdef1234567890abcdef12345678
```

公开地址本身不能直接转走资产；私钥和助记词才是危险信息。

## 5. 新手准备流程

建议按这个顺序准备：

1. 新建一个只用于 Base Sepolia 测试的钱包账户，作为 `HOOK_OWNER`。
2. 再新建一个只用于测试预算的钱包账户，作为 `PROTOCOL_BUDGET_ADDRESS`。
3. 再新建或指定一个只用于测试部署的钱包账户，作为 `DEPLOYER_ADDRESS`。
4. 先不要充值真实资产，只在需要测试网广播时领取少量 Base Sepolia 测试 ETH。
5. 把三个公开地址复制到参数模板，人工检查前 6 位和后 4 位。
6. `CREATE2_DEPLOYER` 由技术侧通过本地模拟或后续测试网 deployer 广播固定，并写入参数模板。

如果你不确定钱包软件怎么操作，先停在第 1 步，不要导入任何已有主钱包。

## 6. 人工复核规则

每次填写地址后，至少做两轮人工复核。每一轮都检查这三项：

- 地址格式：是否以 `0x` 开头，长度是否为 42 个字符。
- 地址用途：尤其检查 `HOOK_OWNER` 和 `PROTOCOL_BUDGET_ADDRESS` 有没有填反。
- 地址片段：前 6 位和后 4 位是否和钱包页面一致。

建议记录成这种格式：

```text
HOOK_OWNER=0x...1234
用途：测试管理员
来源：钱包页面复制
复核人/轮次：第一轮人工复核完成
```

不要只看中间几位，地址中间很长，容易看花。

## 7. 和部署顺序的关系

这些地址准备好后，也还不会立刻部署 Hook。

后续顺序是：

1. 用公开地址跑本地脚本模拟和参数复核。
2. 第一次测试网广播已完成：只部署 `Create2HookDeployer`，得到真实 `CREATE2_DEPLOYER=0xE5Df76A1DF2e959C70c8b7591754a3a2a542Fb6D`。
3. 如果你后续继续明确批准，再部署曲线核心和 `TestnetUsdcAdapter`。
4. 拿到测试网上的 `MOON_TOKEN`、`SUN_CURVE`、`SWAP_ADAPTER`。
5. 用这些地址和固定的 `CREATE2_DEPLOYER` 计算 `HOOK_SALT` 和 `PREDICTED_HOOK`。
6. 预检通过后，再由你单独决定是否广播部署 Hook。

任何一步需要私钥、真实资金或 Base 主网，都停止。

## 8. 当前建议

当前建议只做准备，不进行下一次广播：

- 先确认你是否愿意为测试网新建 2 到 3 个干净钱包账户。
- 不要使用已有主钱包。
- 不要把私钥或助记词发给任何人。
- 当前已收到 3 个测试公开地址，并已完成第一次小额广播部署 `Create2HookDeployer`。下一步先准备和人工复核曲线核心 + `TestnetUsdcAdapter` 的第二次小额广播脚本；没有你的明确批准，不广播。

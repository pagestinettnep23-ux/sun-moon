## 2. 修复项 A：lastMintBlock DoS

影响文件：

```text
contracts/SunCurve.sol
contracts/MoonCurve.sol
test/SunCurve.t.sol
test/MoonCurve.t.sol
```

问题：

```text
原设计使用全局 uint256 lastMintBlock。
如果任意地址在某个区块 mint，可能让同一区块内其他地址 burn 也被 SameBlockMintBurn 阻止。
这会形成跨用户的同区块 DoS 风险。
```

修复：

```text
把全局 lastMintBlock 改为 mapping(address => uint256) public lastMintBlock。

mint / mintFor 成功时记录：
lastMintBlock[payer] = block.number;
lastMintBlock[receiver] = block.number;

burn / burnTo 时只检查：
if (lastMintBlock[payer] == block.number) revert SameBlockMintBurn();
```

保持不变：

```text
不改手续费
不增加最低 mint 金额
不改权限
不改 token
不重构业务逻辑
```

复核要点：

```text
同一付款地址 mint 后同区块 burn 仍会失败
其他地址不再被无关地址的 mint 影响
```

## 3. 修复项 B：Hook 空 hookData 兼容性

影响文件：

```text
contracts/hooks/base/BaseSunMoonUsdcFeeV4Hook.sol
test/hooks/base/BaseSunMoonUsdcFeeV4Hook.t.sol
```

问题：

```text
旧逻辑强制 swap 必须传非空 hookData。
这会影响 Uniswap 前端、聚合器或其他不方便自定义 hookData 的路径。
```

修复：

```text
BaseSunMoonUsdcFeeV4Hook._minUSDCToSunCurve(bytes calldata hookData)

hookData.length == 0 时：
return 1;

hookData 非空且长度小于 32 时：
revert InvalidHookData();

hookData 非空且长度至少 32 时：
继续 abi.decode(hookData, (UsdcFeeHookData));

decode 出来的 minUSDCToSunCurve == 0 时：
revert InvalidMinUSDCToSunCurve();
```

保持不变：

```text
feeToSunCurve == 0 仍然 revert InvalidAmount()
feeToProtocol == 0 仍然 revert InvalidAmount()
不改变 SUN/USDC 2% fee 逻辑
不改变 MOON/USDC 5% fee 逻辑
不改变费用分配比例
不改变权限、token、pool allowlist 或 pause 逻辑
```

已补测试：

```text
空 hookData + 正常金额 swap 成功
空 hookData + 正常金额 swap 触发 FeeRouted
SunCurve 收到注入费用
protocolBudget 收到协议费用
非空 hookData 且 minUSDCToSunCurve = 0 仍然失败
minUSDCToSunCurve 大于实际注入金额仍然失败
极小额 swap 导致费用为 0 仍然失败
短的非空 hookData 失败，错误为 InvalidHookData
```

专项测试结果：

```text
forge test --match-path test\hooks\base\BaseSunMoonUsdcFeeV4Hook.t.sol
结果：18 passed, 0 failed
```

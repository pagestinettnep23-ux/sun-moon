状态含义：

```text

```

## 4. H-02：afterSwap delta sign

来源：

```text
其他 AI / 审计复核提出：
BaseSunMoonUsdcFeeV4Hook.afterSwap 返回正 delta，可能方向错误，可能导致双重收费。
```

AI 给出的风险等级：

```text
High
```

当前状态：

```text
disputed / not confirmed
```

我们的判断：

```text
暂不确认为 confirmed high。
```

理由：

```text
Uniswap v4 中，afterSwap 返回正 delta 表示 Hook 在 unspecified token 上拿取资产。
当前 Hook 使用 poolManager.take(...) 收 USDC，再立即路由到 SunCurve 和 protocolBudget。
这是 v4 自定义会计里的正常收费路径。
```

已补测试：

```text
testExactOutputBuySunWithUsdcInputChargesAfterSwapFeeOnce
testExactOutputBuyMoonWithUsdcInputChargesAfterSwapFeeOnce
```

测试覆盖：

```text
用户用 USDC 作为输入，exact output 买 SUN。
用户用 USDC 作为输入，exact output 买 MOON。
SunCurve 收费正确。
protocolBudget 收费正确。
Hook USDC 余额最终为 0。
用户实际 USDC 支出 = 基础 USDC 输入 + 单次 Hook fee。
没有复现双重收费。
```

测试结果：

## 6. LMB-01：lastMintBlock DoS

来源：

```text
审计 / 工具复核发现：
SunCurve 和 MoonCurve 原来使用全局 lastMintBlock。
任意用户 mint 后，可能影响其他用户同区块 burn。
```

当前状态：

```text
fixed
```

修复内容：

```text
把全局 uint256 lastMintBlock 改成 mapping(address => uint256) public lastMintBlock。
mint / mintFor 成功时记录 payer 和 receiver。
burn / burnTo 只检查 payer 自己的 lastMintBlock。
```

影响文件：

```text
contracts/SunCurve.sol
contracts/MoonCurve.sol
test/SunCurve.t.sol
test/MoonCurve.t.sol
```

证据：

```text

commit=8d7ccf3
docs/Base-主网审计修复记录-2026-05-19.md
```

测试结果：

```text
forge test --threads 1 --isolate
366 tests passed, 0 failed, 0 skipped
```

下一步：

```text
保持当前修复。
如果其他 AI 对该项继续提问，让它基于 rc4 或更新 tag 复核。
```

本轮复核：

```text
contracts/SunCurve.sol 和 contracts/MoonCurve.sol 仍使用 mapping(address => uint256) public lastMintBlock。
testSunBurnAllowsDifferentPayerWhenAnotherUserMintedSameBlock 通过。
testMoonBurnAllowsDifferentPayerWhenAnotherUserMintedSameBlock 通过。
说明：一个用户 mint 不会再卡住另一个用户同区块 burn。
```

## 7. HD-01：空 hookData 兼容性

来源：

```text
发现 BaseSunMoonUsdcFeeV4Hook 强制要求非空 hookData。
这会影响 Uniswap 前端、聚合器或其他不方便传自定义 hookData 的 swap 路径。
```

当前状态：

```text
fixed
```

修复内容：

```text
hookData.length == 0 时 return 1。
hookData 非空时继续 abi.decode。
decode 出 minUSDCToSunCurve == 0 仍然 revert。
feeToSunCurve 或 feeToProtocol 为 0 的极小额 swap 仍然 revert。
```

# Base 主网逐笔广播草案 - 2026-05-20

本文档只是“上线前逐笔广播顺序草案”，不是主网执行批准。

当前仍然遵守硬性要求：

- 不执行 Base 主网交易。
- 不使用真实资金。
- 不要求、记录、粘贴私钥、助记词、恢复词。
- 不使用 `--private-key`。
- 不把 dry-run 当成已经上线。
- 不把预测地址当成已经部署地址。

重要提醒：真实 Base 主网广播一定会消耗真实 Base ETH gas。只要“不得使用真实资金”这条要求仍然有效，就只能做草案、只读检查、fork dry-run 和测试，不能进入真实广播。

## 0. 当前结论

当前主网部署路径只允许使用新版：

```text
BaseSunMoonUsdcFeeV4Hook
```

当前仓库里已有的是主网 dry-run / 计算脚本，还没有可以直接执行真实主网广播的最终脚本。

所以本文的用途是：

1. 把将来每一笔主网交易的顺序写清楚。
2. 写清楚每一笔交易前后要检查什么。
3. 帮你避免误用旧 Hook、错地址、错网络、漏绑定。

## 1. 广播前总停止条件

出现任意一条，立即停止：

- 钱包网络不是 Base mainnet `8453`。
- 任何脚本、命令或页面要求输入私钥、助记词、恢复词。
- 任何命令出现 `--private-key`。
- 任何输出显示 `BaseMoonAmmFeeV4Hook` 被用于主网默认路径。
- 任何输出显示 `broadcastRequested=true`，但当前还不是最终批准广播阶段。
- `USDC_TOKEN` 不是 `0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913`。
- `USDC_DECIMALS` 不是 `6`。
- `MAINNET_DEPLOYER` nonce 变化导致核心预测地址变化。
- `PREDICTED_HOOK`、`HOOK_SALT` 或两个 `poolId` 和最终 dry-run 不一致。
- 19 笔草案没有显示 `totalTransactionsPlanned=19`、`moonAmmBindingTxIncluded=true`、`broadcastAllowed=false`、`executionBlocked=true`。
- 任意主网流程引用旧 Hook、legacy 脚本或测试网脚本，例如 `BaseMoonAmmFeeV4Hook`、`FindBaseMoonAmmFeeV4HookSalt`、`PrepareBaseSepoliaHookDeploy`、`TinyMoonUsdc`。
- `SunCurve.usdt()`、`Hook.usdc()`、Base 官方 USDC 三者有任何一个不一致。
- 官方前端或文档把无 Hook 池当成官方池。
- 测试或 fork dry-run 有失败。

- 最终上线前必须有 19 笔只读广播草案覆盖 `SunCurve.setMoonAMM(PREDICTED_HOOK)`，并在真实执行后只读检查 `SunCurve.moonAMM() == PREDICTED_HOOK`。

当前提醒：现有 `PrepareBaseMainnetSunMoonUsdcForkDryRun.s.sol` 仍然只模拟 Hook 部署、两个池白名单、两个池初始化和 Hook renounce，输出 `transactionsPlanned=6`。它是 fork 安全演练，不是 19 笔广播草案。第 16 笔绑定已经由 `PrepareBaseMainnetSunMoonUsdcBroadcastDraft.s.sol` 纳入只读 19 笔计划；正式广播前仍必须在每一步执行后做只读检查，尤其确认 `SunCurve.moonAMM() == PREDICTED_HOOK`。

## 4. 逐笔广播草案总览

按当前 dry-run 和补齐的绑定动作，主网第一批草案预计是 19 笔：

- 核心部署与基础配置：12 笔。
- Hook、白名单、绑定、池初始化、Hook 锁权限：7 笔。

其中第 16 笔 `SunCurve.setMoonAMM(PREDICTED_HOOK)` 是必须补进真实执行顺序的关键绑定。

当前提醒：如果 fork dry-run 显示 `transactionsPlanned=6`，这是正常的 fork 演练结果；19 笔只读广播草案以 `PrepareBaseMainnetSunMoonUsdcBroadcastDraft.s.sol` 输出为准。只要未进入单独批准的真实广播阶段，这份草案仍只能用于人工复核。

注意：`SunCurve` / `MoonCurve` / Token owner 的最终 renounce 不放在这 19 笔里。它们需要另写 renounce-only 预检和测试，单独决定。

R-08 说明：官方只认上面两个带 `BaseSunMoonUsdcFeeV4Hook` 的官方池。别人创建无 Hook 池是开放市场事实，不是官方路径；官方前端、公告、教程、部署清单都不能把无 Hook 池当成官方池。

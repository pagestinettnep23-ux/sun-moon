## 使用规则

- 这里只能填写公开地址和公开配置。
- 不允许填写私钥、助记词、恢复词、硬件钱包 PIN、完整带密钥的 RPC URL。
- `poolId`、CREATE2 Hook 地址、`sqrtPriceX96` 这类值必须由脚本计算，不能手填猜测。
- 每一项都必须有状态：`待填`、`已复核`、`不通过`。
- 任意一项为 `不通过`，停止后续流程。
- 任何广播交易都必须单独列出交易草案，并由用户再次明确批准。

## 8. 计算脚本

已经完成的本地脚本：

```powershell
forge script script/PrepareBaseMainnetCoreDeployDryRun.s.sol
forge script script/ComputeBaseSunMoonUsdcPoolIds.s.sol
```

`PrepareBaseMainnetCoreDeployDryRun.s.sol` 只预测核心合约 CREATE 地址并模拟配置，不会广播。

`ComputeBaseSunMoonUsdcPoolIds.s.sol` 只计算 `PoolKey -> poolId`，不会广播，不会加白名单，不会初始化池。

正式参数准备好后，才允许用公开地址运行：

```text
HOOK_ADDRESS=
SUN_TOKEN=
MOON_TOKEN=
USDC_TOKEN=0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913
SUN_USDC_POOL_FEE=3000
SUN_USDC_POOL_TICK_SPACING=60
MOON_USDC_POOL_FEE=3000
MOON_USDC_POOL_TICK_SPACING=60
```

运行前必须确认：

- `HOOK_ADDRESS` 是正式预测或已部署的新统一 Hook。
- `SUN_TOKEN` 和 `MOON_TOKEN` 是正式主网地址。
- `USDC_TOKEN` 是 Circle 官方 Base USDC。
- fee / tickSpacing 是用户已确认的池参数。

## 9. 白名单与 renounce 顺序

计划顺序必须是：

1. 用正式参数计算 `SUN_USDC_POOL_ID` 和 `MOON_USDC_POOL_ID`。
2. 配置 Hook 白名单：
   - `setAllowedSunUsdcPool(SUN_USDC_POOL_ID, true)`
   - `setAllowedMoonUsdcPool(MOON_USDC_POOL_ID, true)`
3. 配置 `protocolBudget`、adapter、`SunCurve.moonAMM` 等必要绑定。
4. 用只读调用复核全部参数。
5. 执行 `renounceOwnership()` 或等效放弃管理权。
6. 再次复核 owner 已放弃。

不允许：

- renounce 后再尝试新增池白名单。
- 白名单任何 `SUN/WETH`、`MOON/WETH`、`SUN/ETH`、`MOON/ETH` 项目支持池。
- 把第三方池描述成协议支持路径。

## 11. 停止条件

出现任意一条，立即停止：

- 有人要求提供私钥、助记词或恢复词。
- 有人要求直接主网广播。
- 发现 Base Sepolia 地址混入 Base Mainnet 参数。
- 官方地址复核和本地代码常量不一致。
- `PROTOCOL_BUDGET_WALLET` 被填成任何 owner 或管理员。
- poolId 不是由 `PoolKey.toId()` 计算。
- Hook 地址低 14 位权限 bit 不是 `204`。
- mainnet fork dry-run 没有通过。

## 2. 覆盖重点

`SunCurve` fuzz：

```text
随机 USDT mint 后，SUN 输出、储备、协议费用和 USDT 余额必须一致
随机 SUN burn 后，USDT 输出、储备减少、协议费用和总供应量必须一致
连续 mint 后 SUN 协议价格不能下降
SUN 自由转账不能改变曲线储备、总供应量或协议价格
授权 Hook 注入 USDT 后，只增加储备，不改变 SUN 总供应量
```

`MoonCurve` fuzz：

```text
随机 SUN 输入 mint MOON 后，MOON 输出、SUN 储备、协议费用和 SUN burnAndRetain 必须一致
随机 MOON burn 后，SUN 输出、SUN 储备、MOON 供应量和费用分配必须一致
MOON 自由转账不能改变 MoonCurve 储备或公平供应量
```

`SunCurve` invariant：

```text
任意顺序组合 mint / burn / inject / SUN transfer
始终要求 curveReserve == USDT balance
始终要求 getSunPrice() == curveReserve / totalSupply
```

`MoonCurve` invariant：

```text
任意顺序组合 SUN mint / MOON mint / MOON burn / SUN transfer / MOON transfer
始终要求 SunCurve curveReserve == USDT balance
始终要求 MoonCurve sunReserve == SUN balance
始终要求 MOON totalSupply 贴近 currentFairSupply
始终要求 SunCurve 价格公式一致
```

`BaseSunMoonUsdcFeeV4Hook` fuzz：

```text
随机 SUN/USDC 指定 USDC 输入时，始终按 1.5% 给 SunCurve、0.5% 给协议经费
随机 MOON/USDC 指定 USDC 输入时，始终按 3% 给 SunCurve、2% 给协议经费
renounce 后任意调用者都不能新增白名单、改协议经费、暂停或转移 owner
```

## 4. 发现的边界说明

新增 fuzz 过程中确认了两个整数舍入边界：

```text
MOON burn 极小数量时，按 3% / 2% 计算出的手续费可能为 0，合约会按 InvalidAmount 拒绝。
极小金额 burnAndRetain 后，SUN 协议价格可能因为整数舍入保持不变，而不是严格上涨。
```

当前处理：

```text
成功路径 fuzz 使用有意义的最小金额，避免把 dust 金额当成正常业务路径。
价格断言使用“不能下降”，而不是“必须严格上涨”。
```

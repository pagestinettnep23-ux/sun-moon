# 本地部署说明

本文档记录当前本地部署脚本的用途和运行方式。

## 当前脚本

脚本文件：

```text
script/DeployLocal.s.sol
```

部署顺序：

```text
MockUSDT
  -> SunToken
  -> SunCurve
  -> MoonToken
  -> MoonCurve
```

权限绑定：

```text
SunToken.setMinter(SunCurve)
SunCurve.setMoonCurve(MoonCurve)
SunCurve.setMoonAMM(MOON_AMM_ADDRESS)
MoonToken.setMinter(MoonCurve)
```

## 默认参数

```text
MockUSDT decimals = 6
SUN_MAX_MINT_USDT = 10,000 USDT
MOON_K = 5,000,000 MOON
MOON_S = 1,200,000 SUN
MOON_MAX_MINT_USDT_EQUIV = 10,000 USDT
MOON_LAUNCH_DELAY = 0
```

## 本地模拟运行

只验证部署流程，不真正发交易：

```powershell
forge script script/DeployLocal.s.sol:DeployLocal -vvv
```

当前已验证通过。

## 本地链广播

后续如果启动 Anvil 本地区块链，可以用：

```powershell
anvil
```

然后在另一个 PowerShell 中运行：

```powershell
forge script script/DeployLocal.s.sol:DeployLocal --rpc-url http://127.0.0.1:8545 --broadcast
```

如果需要指定部署钱包，再通过 `.env` 设置 `PRIVATE_KEY`，不要把真实私钥写进代码或文档。

## 可选环境变量

```text
PRIVATE_KEY=
PROTOCOL_BUDGET_ADDRESS=
MOON_AMM_ADDRESS=
MOON_LAUNCH_DELAY=0
```

如果不设置：

- `PROTOCOL_BUDGET_ADDRESS` 默认使用部署者地址。
- `MOON_AMM_ADDRESS` 默认使用部署者地址。
- `MOON_LAUNCH_DELAY` 默认为 `0`。

## 安全提醒

- 当前脚本只用于本地和测试用途。
- 主网部署前必须单独确认真实参数。
- 主网部署前必须记录合约地址、构造参数和权限绑定结果。
- 不要使用真实主网私钥做本地测试。

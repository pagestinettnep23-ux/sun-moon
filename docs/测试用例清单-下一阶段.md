# 测试用例清单 - 下一阶段

本文档整理 SUN/MOON 下一阶段建议测试用例。当前目标是围绕前端交互、本地链流程、合约调用边界和后续 Hook/AMM 方案做测试规划，不涉及真实资金建议，不修改合约代码和部署脚本。

2026-05-15 主网决策更新：SUN 永久不做官方 AMM 流动性，SUN 加池守卫相关用例只作为历史原型记录，不进入主网测试计划。

## 1. 测试范围说明

下一阶段建议重点测试：

- 前端页面能否正确读取用户余额、价格、储备和倒计时。
- 用户能否按正确路径完成 `USDT -> SUN -> MOON`。
- 用户能否按正确路径完成 `MOON -> SUN -> USDT`。
- 授权、余额不足、超过上限、未启动等错误状态是否能正确提示。
- 本地链 MockUSDT 流程是否清晰。
- 后续 Hook/AMM 功能进入开发前，需要明确哪些测试必须补上。

下一阶段暂不测试：

- 主网真实资金。
- 真实 USDT。
- 收益承诺或投资建议。
- 未完成的 AMM 加池真实交易。
- 未完成的 Hook 生产级部署。

## 12. 建议执行顺序

建议下一阶段按以下顺序推进测试：

1. 先跑通前端读取数据。
2. 再跑通本地链 MockUSDT 领取。
3. 再跑通 SUN Mint/Burn。
4. 再跑通 MOON Mint/Burn。
5. 再做完整用户路径测试。
6. 最后补错误状态和边界测试。
7. Hook/AMM 测试等对应功能进入开发后再执行。

## 13. 验收标准

下一阶段可以认为通过的标准：

- 页面能正确连接钱包和识别网络。
- 页面能显示用户 USDT、SUN、MOON 余额。
- 页面能显示 SUN 和 MOON 的关键曲线数据。
- 用户能在本地链完成 `MockUSDT -> SUN -> MOON`。
- 用户能在本地链完成 `MOON -> SUN -> USDT`。
- 未授权、余额不足、超过上限、MOON 未启动等状态都有明确提示。
- 测试过程不接触真实资金。
- 不修改合约代码和部署脚本。

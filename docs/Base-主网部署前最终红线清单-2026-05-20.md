# Base 主网部署前最终红线清单 - 2026-05-20

本文是给技术小白使用的一页停止清单。

它不是主网部署批准，不是广播记录，也不是交易教程。

只要本清单里任何一条不满足，就停止，不要继续。

```text
不执行 Base 主网交易
不使用真实资金
不要求或记录私钥、助记词、恢复词
不使用 --private-key
不广播
```

## 1. 最高优先级红线

看到以下任意情况，立即停止：

```text
任何人要求你输入私钥
任何人要求你输入助记词
任何人要求你输入恢复词
任何人要求你粘贴钱包导出内容
任何命令包含 --private-key
任何命令包含 --broadcast
任何命令包含 cast send
任何步骤要求使用真实 ETH 或真实 USDC 做演练
任何页面弹出 Base 主网交易签名
任何脚本显示 broadcastRequested=true
任何脚本显示 executeRequested=true
任何脚本显示 privateKeyPresent=true
任何脚本显示 broadcastAllowed=true
任何脚本显示 executionBlocked=false
```

当前硬性要求仍然是：

```text
不执行 Base 主网交易。
不使用真实资金。
```

所以当前阶段只能做：

```text
文档
本地测试
只读检查
fork dry-run
Git 提交 / 推送 / 标签
```

## 2. 网络和钱包红线

看到以下任意情况，立即停止：

```text
钱包网络显示 Base mainnet，并且页面要求确认交易
钱包提示会消耗真实 gas
钱包提示会发送真实 ETH
钱包提示会转出真实 USDC
钱包提示签名内容看不懂
浏览器页面要求连接真实资金钱包做演练
```

允许继续的情况只有：

```text
命令是只读检查
命令是本地测试
命令是 fork dry-run
命令没有 --broadcast
命令没有 --private-key
没有钱包弹窗
没有真实资金提示
```

## 3. Git 版本红线

```text

工作区没有未提交的 test/ 主网测试改动
```

如果改过以下目录，必须停止并重新跑测试：

```text

script/
test/
foundry.toml

```

```text

```

如果本地 HEAD 不是这个提交或更新后的明确安全提交，不要直接进入下一步。


R-04 后继续只读复核时，旧 Hook / 旧 poolId 不能再当最终值使用。

## 6. Hook 路径红线

Base 主网路径只允许新版 Hook：

```text
BaseSunMoonUsdcFeeV4Hook
```

看到以下任意旧路径用于主网，立即停止：

```text
BaseMoonAmmFeeV4Hook
FindBaseMoonAmmFeeV4HookSalt
PrepareBaseSepoliaHookDeploy
TinyMoonUsdc
```

旧路径可以留在历史文档或测试里，但不能成为主网默认路径。

旧 Hook / legacy 脚本必须标废。它们可以存在于历史记录里，但如果真实主网命令、最终部署清单、前端配置或公告里引用它们，立即停止。

## 9. 绑定红线

19 笔草案必须包含：

```text
SunCurve.setMoonAMM(PREDICTED_HOOK)
```

执行 `SunCurve.setMoonAMM(PREDICTED_HOOK)` 前，必须只读确认：

```text
code(PREDICTED_HOOK) 存在
实际 Hook 地址 == PREDICTED_HOOK
actualLow14Bits == 204
Hook.usdc() == USDC_TOKEN
SunCurve.usdt() == USDC_TOKEN
Hook.usdc() == SunCurve.usdt()
Hook.sunCurve() == SUN_CURVE
SunCurve.moonCurve() == MOON_CURVE
SunCurve.moonAMM() == address(0)
```

真实部署后才可以做以下只读检查：

```text
SunCurve.moonAMM() == PREDICTED_HOOK
Hook.usdc() == USDC_TOKEN
Hook.sunCurve() == SUN_CURVE
Hook.protocolBudget() == PROTOCOL_BUDGET_WALLET
```

没有真实部署前，不允许把这些项打勾成“已链上通过”。

## 10. 官方池和挂起项红线

官方只认带 `BaseSunMoonUsdcFeeV4Hook` 的官方池。别人创建无 Hook 池是市场事实，不是合约漏洞；但官方前端、文档、公告不能把无 Hook 池当成官方池。

以下核心逻辑暂时挂起，不建议临上线改：

```text
SunCurve / MoonCurve mint、burn 没有 minOut 滑点保护
MoonCurveMath.burnQuote 极端精度下溢防御上限
S 参数是否改小
```

如果要处理这些项，必须单独开设计、测试、审计和 dry-run，不要在 R-04 修复后顺手改。

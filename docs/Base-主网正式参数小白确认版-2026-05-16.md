## 2. 你以后只需要提供什么

以后如果进入“主网部署草案”阶段，你只需要提供公开信息：

| 你提供的东西 | 是什么 | 能不能公开 |
| --- | --- | --- |
| `MAINNET_ADMIN_WALLET` | 临时管理员普通钱包公开地址 | 可以公开 |
| `PROTOCOL_BUDGET_WALLET` | 协议经费收款普通钱包公开地址 | 可以公开 |
| 前端复核人 | 谁负责复核主网地址 | 可以公开姓名或内部代号 |

你永远不要提供：

| 不能提供的东西 | 原因 |
| --- | --- |
| 私钥 | 拿到私钥的人可以直接控制钱包 |
| 助记词 / 恢复词 | 等于整个钱包控制权 |
| 硬件钱包 PIN | 不应该出现在聊天或文档里 |
| 带密钥的完整 RPC URL | 可能泄露付费账户或调用额度 |

## 4. SUN/MOON 池子的最终方向

已经确认的方向：

| 项目 | 结论 |
| --- | --- |
| SUN 能不能自由转账 | 能 |
| MOON 能不能自由转账 | 能 |
| 市场能不能自己建池 | 能，协议不强行禁止 |

## 7. 哪些值必须由脚本算

这些值不能手填、不能猜：

| 值 | 为什么不能手填 |
| --- | --- |
| `SUN_TOKEN / MOON_TOKEN / 曲线合约地址` | 这些地址受部署钱包 nonce 和部署顺序影响，必须由 dry-run 或正式部署输出 |
| `CREATE2_HOOK_DEPLOYER` | 它也受部署钱包 nonce 和部署顺序影响，必须由脚本预测或正式部署输出 |
| `PREDICTED_HOOK` | 它由 CREATE2 部署工具、salt、合约字节码一起算出来 |
| `HOOK_SALT` | 必须搜索出满足 v4 权限 bit 的 salt |
| `SUN_USDC_POOL_ID` | 必须由完整 `PoolKey.toId()` 计算 |
| `MOON_USDC_POOL_ID` | 必须由完整 `PoolKey.toId()` 计算 |
| `INITIAL_TICK` | 必须由初始价格计算 |
| `SQRT_PRICE_X96` | 必须由初始价格计算，是初始化池子的输入 |

你只负责确认输入和结果看起来合理；脚本负责计算。

## 8. 已决定的初始价格

创建 v4 池时，要给一个初始价格。

这件事不能随便填，因为它会影响刚开始的池子价格。

owner 已确认：按照两个代币的初始铸造价格来设定同样的 AMM 初始化价格。

```text
SUN/USDC 初始价格：1 SUN = 1 USDC
MOON/USDC 初始价格：1 MOON = 0.24 USDC
```

这里用的是曲线初始报价口径：SUN 初始净铸造报价是 `1 USDC/SUN`；MOON 初始铸造价是 `0.24 SUN/MOON`，再按 `1 SUN = 1 USDC` 换算成 `0.24 USDC/MOON`。

脚本后续还要根据正式 token 地址排序，把这两个价格换算成 Uniswap v4 需要的 `INITIAL_TICK` 和 `SQRT_PRICE_X96`。

## 9. owner 放弃是什么意思

上线配置期需要临时管理员，是为了完成这些动作：

- 设置协议经费地址。
- 设置支持的 `SUN/USDC` poolId。
- 设置支持的 `MOON/USDC` poolId。
- 绑定 Hook 和曲线。

全部配置完成并复核后，要执行：

```text
renounceOwnership()
```

人话解释：

```text
放弃管理员权限。
以后不能再改支持池。
以后不能再改协议经费地址。
以后不能再暂停或恢复。
```

这符合你的目标：上线后项目方只保留协议经费收款地址，不保留日常管理权。

缺点也必须写清楚：

```text
放弃后如果参数配错，不能再靠管理员修。
所以放弃前必须反复 dry-run、复核、外部 review。
```

# 给主开发 AI 的任务指令

本文档用于直接交给负责主要开发工作的 AI 智能体。请严格按照本文档执行，不要超出当前阶段范围。

## 1. 当前任务目标

请基于现有 SUN/MOON 项目文档，进入下一阶段：前端第一版开发准备和实现。

当前阶段目标是：

```text
做一个本地链/测试环境可用的前端交互页面
让用户能看懂当前数据
让用户能完成核心测试路径
```

核心测试路径：

```text
MockUSDT -> SUN -> MOON
MOON -> SUN -> USDT
```

- 新增前端按钮和交互状态。

## 4. 禁止做的事情

当前阶段禁止：

- 不要修改 Solidity 合约代码。
- 不要修改部署脚本。
- 不要部署主网。
- 不要接触真实资金。
- 不要接入真实 USDT。
- 不要提供投资建议。
- 不要写收益承诺文案。
- 不要做一键 `USDT -> MOON`。
- 不要做一键 `MOON -> USDT`。
- 不要进入生产级 Hook/AMM 阶段。
- 不要实现复杂 AMM 加池入口。
- 不要绕过 `SUN -> MOON` 的既定路径。

如果发现必须修改合约或部署脚本才能继续，请先停止并输出原因，不要自行修改。

## 5. 前端第一版功能范围

前端第一版只需要做一个简单可用页面，不需要复杂 UI。

页面建议包含四个区域：

```text
顶部状态区
数据总览区
核心操作区
本地测试区
```

### 5.1 顶部状态区

需要显示：

- 当前钱包连接状态。
- 当前钱包地址。
- 当前网络。
- Chain ID。

需要按钮：

- `连接钱包`
- `断开连接`
- `切换网络`
- `刷新数据`

### 5.2 数据总览区

需要显示：

- 用户 USDT 余额。
- 用户 SUN 余额。
- 用户 MOON 余额。
- SUN 当前曲线价格。
- MOON 当前 Mint 价格。
- MOON 当前 Burn 价格。
- SUN 总供应量。
- MOON 当前供应量。
- SUN 储备 USDT。
- MOON 储备 SUN。
- MOON 启动时间。
- MOON 启动倒计时。
- 协议预算钱包地址。

价格说明：

- SUN 价格只能读取曲线账本。
- MOON 价格应基于曲线价格，不要用 AMM 市场价格代替。

### 5.3 核心操作区

需要支持四个操作：

#### 买 SUN

路径：

```text
USDT -> SUN
```

需要按钮：

- `授权 USDT`
- `铸造 SUN`

需要显示：

- 输入 USDT 数量。
- 预计获得 SUN。
- 单笔上限 `10,000 USDT`。
- 手续费说明：SUN Mint 总手续费 2%。

#### 卖 SUN

路径：

```text
SUN -> USDT
```

需要按钮：

- `授权 SUN`
- `卖出 SUN`

需要显示：

- 输入 SUN 数量。
- 预计取回 USDT。
- 手续费说明：SUN Burn 总手续费 2%。

#### 铸造 MOON

路径：

```text
SUN -> MOON
```

需要按钮：

- `授权 SUN`
- `铸造 MOON`

需要显示：

- 输入 SUN 数量。
- 当前输入 SUN 折算的 USDT 等值。
- 预计获得 MOON。
- 单笔上限：等值 `10,000 USDT` 的 SUN。
- 手续费说明：MOON Mint 总手续费 5%。
- 如果 MOON 未启动，显示倒计时并禁用按钮。

#### 卖 MOON

路径：

```text
MOON -> SUN
```

需要按钮：

- `卖出 MOON`

需要显示：

- 输入 MOON 数量。
- 预计取回 SUN。
- 手续费说明：MOON Burn 总手续费 5%。
- 当前最小版合约中，`MoonCurve.burn()` 由曲线合约直接销毁用户 MOON，不需要用户提前授权 MOON。

### 5.4 本地测试区

仅本地链或测试环境显示：

- `领取 MockUSDT`
- MockUSDT 地址。
- SunToken 地址。
- SunCurve 地址。
- MoonToken 地址。
- MoonCurve 地址。
- 本地链说明。

提示文案：

```text
当前为本地测试环境，MockUSDT 和测试账户只能用于开发测试，不能用于真实资金。
```

## 6. 必须处理的前端状态

必须处理：

- 钱包未连接。
- 当前网络错误。
- 合约地址缺失。
- 余额不足。
- 授权不足。
- 输入为空。
- 输入为 0。
- 输入超过单笔上限。
- MOON 尚未启动。
- 交易等待确认。
- 交易成功。
- 交易失败。
- 合约读取失败。

按钮禁用规则：

- 状态不满足时，不允许用户提交交易。
- 交易确认中时，避免重复点击。
- 错误原因需要尽量用中文显示。

## 7. 必须遵守的用户路径

必须遵守：

```text
USDT -> SUN -> MOON
MOON -> SUN -> USDT
```

不要提供：

```text
USDT -> MOON
MOON -> USDT
```

可以在页面中说明：

```text
如果想从 MOON 退出到 USDT，请先卖出 MOON 得到 SUN，再卖出 SUN 得到 USDT。
```

但不要把它做成一键操作。

## 8. 建议测试清单

开发完成后，请至少验证：

- 钱包未连接时按钮禁用。
- 错误网络时提示切换网络。
- 用户能领取 MockUSDT。
- 用户能授权 USDT。
- 用户能用 USDT 铸造 SUN。
- 用户能授权 SUN。
- 用户能用 SUN 铸造 MOON。
- 用户能卖出 MOON 得到 SUN。当前最小版合约中，卖出 MOON 不需要提前授权 MOON。
- 用户能卖出 SUN 得到 USDT。
- 输入 0 时按钮禁用。
- 余额不足时提示错误。
- SUN Mint 超过 `10,000 USDT` 时提示错误。
- MOON Mint 超过等值 `10,000 USDT` 时提示错误。
- MOON 未启动时不能铸造。
- 每次交易成功后页面数据刷新。

更完整测试项请参考：

```text
docs/测试用例清单-下一阶段.md
```

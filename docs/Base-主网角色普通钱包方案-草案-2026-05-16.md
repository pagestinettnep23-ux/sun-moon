# Base 主网角色普通钱包方案草案 - 2026-05-16

本文档记录 owner 最新决定：主网角色地址不使用多签钱包，改用普通钱包公开地址。

这不是主网部署批准。当前仍然不部署 Base 主网，不广播主网交易，不接真实资金，不收集私钥。

## 1. 核心决定

```text
不用多签钱包
使用普通钱包公开地址
只填写公开地址
永远不在文档、聊天或代码里填写私钥、助记词、恢复词
```

普通钱包比多签简单，但安全风险更集中：只要私钥或助记词泄露，钱包就会被完全控制。

## 2. 角色地址

| 角色 | 钱包类型 | 用途 | 是否长期保留权力 |
| --- | --- | --- | --- |
| `MAINNET_DEPLOYER` | 普通钱包 | 部署合约和支付 gas | 不保留 |
| `MAINNET_ADMIN_WALLET` | 普通钱包 | 上线配置期临时管理员 | 配置后放弃 owner |
| `PROTOCOL_BUDGET_WALLET` | 普通钱包 | 接收协议经费 USDC | 只收款，不管理 |
| `CREATE2_DEPLOYER_OWNER` | 普通钱包 | 临时控制 CREATE2 部署工具 | 不作为长期权限 |

## 3. 硬性隔离规则

```text
PROTOCOL_BUDGET_WALLET 不能等于 owner
PROTOCOL_BUDGET_WALLET 不能等于 deployer
PROTOCOL_BUDGET_WALLET 不能等于 adapter owner
PROTOCOL_BUDGET_WALLET 不能等于 pause controller
PROTOCOL_BUDGET_WALLET 不能等于 CREATE2_DEPLOYER_OWNER
MAINNET_ADMIN_WALLET 和 PROTOCOL_BUDGET_WALLET 不建议相同
不要复用 Base Sepolia 测试钱包作为主网正式钱包
```

## 4. Hook 构造参数对应关系

```text
protocolBudget = PROTOCOL_BUDGET_WALLET
initialOwner = MAINNET_ADMIN_WALLET
```

`MAINNET_ADMIN_WALLET` 只是配置期 owner。配置完成、白名单和收款地址复核通过后，必须执行 `renounceOwnership()` 或进入等效不可管理状态。

## 6. 停止条件

出现任何一条，立即停止：

- 有人要求你发私钥、助记词或恢复词。
- 有人要求把私钥写进 `.env`、脚本、文档或聊天。
- 有人要求直接主网广播。
- 有人要求用测试网钱包地址顶替主网正式钱包。
- 有人要求让 `PROTOCOL_BUDGET_WALLET` 同时做管理员或 owner。

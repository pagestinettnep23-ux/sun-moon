## 2. 本轮必须纳入审计的修复范围

核心合约：

```text
contracts/SunCurve.sol
contracts/MoonCurve.sol
contracts/hooks/base/BaseSunMoonUsdcFeeV4Hook.sol
```

直接相关测试：

```text
test/SunCurve.t.sol
test/MoonCurve.t.sol
test/hooks/base/BaseSunMoonUsdcFeeV4Hook.t.sol
```

本轮修复摘要：

```text
SunCurve / MoonCurve：
把全局 lastMintBlock 改为 mapping(address => uint256) public lastMintBlock。
mint / mintFor 成功时记录 payer 和 receiver。
burn / burnTo 只检查 payer 自己的 lastMintBlock。

BaseSunMoonUsdcFeeV4Hook：
hookData.length == 0 时默认 minUSDCToSunCurve = 1。
非空 hookData 继续 decode。
minUSDCToSunCurve == 0 仍然失败。
feeToSunCurve 或 feeToProtocol 为 0 的极小额 swap 仍然失败。
```

## 9. 上线前人工硬闸门

以下三项必须在上线前检查清单中保留：

```text
SunCurve.usdt() == USDC_TOKEN
BaseSunMoonUsdcFeeV4Hook.usdc() == USDC_TOKEN
SunCurve.moonAMM() == Hook
```

以下情况出现任意一条，立即停止：

```text
有人要求提供私钥、助记词或恢复词
有人要求使用 --private-key
有人要求在未完成最终审计和人工复核前执行 Base 主网广播
有人要求用 Base Sepolia 地址替代 Base 主网地址
PREDICTED_HOOK 低 14 位不是 204
PREDICTED_HOOK 和实际部署 Hook 地址不一致
SunCurve.usdt()、Hook.usdc()、SunCurve.moonAMM() 三项硬闸门不通过
```

# 前端交互说明

本文档面向前端开发和项目协作人员，用小白能看懂的方式说明 SUN/MOON 当前机制、前端第一版需要展示的数据、需要提供的按钮，以及暂时不做的功能边界。

## 1. 项目当前进展

SUN/MOON 是一个双代币机制：

- `SUN` 是基础储备代币，用户用 `USDT` 铸造 SUN，也可以销毁 SUN 换回 USDT。
- `MOON` 是稀缺飞轮代币，用户用 `SUN` 铸造 MOON，也可以销毁 MOON 换回 SUN。
- 用户不能直接用 USDT 一键买 MOON，必须先获得 SUN，再用 SUN 铸造 MOON。
- SUN 的曲线价格通过储备和手续费设计，目标是曲线价格不下降。
- MOON 使用指数曲线，越往后铸造成本越高。
- MOON 的手续费会反过来推高 SUN 曲线价格，从而让 SUN 和 MOON 形成飞轮关系。

当前已经完成的阶段性工作：

- 已完成 Foundry 本地开发环境。
- 已完成 `MockUSDT`、`SunToken`、`SunCurve`、`MoonToken`、`MoonCurve` 的最小版本。
- 已完成 SUN 和 MOON 的数学测试。
- 已完成真实合约版 SUN/MOON 测试。
- 已完成安全边界测试。
- 已完成本地 Anvil 部署脚本。
- 已在 Anvil 本地链完成一次完整交互测试。

本地链已验证的用户路径：

```text
领取 MockUSDT
  -> 用 USDT 铸造 SUN
  -> 用 SUN 铸造 MOON
  -> 卖回部分 MOON 得到 SUN
  -> 卖回部分 SUN 得到 USDT
```

当前阶段的重点不是主网上线，而是把本地链和测试网交互流程整理清楚，方便后续前端开发和测试。

## 2. 用户主要操作路径

### 2.1 买 SUN

用户流程：

```text
USDT -> 授权 SunCurve -> 铸造 SUN
```

前端说明：

- 用户输入想花费的 USDT 数量。
- 前端显示预计可获得多少 SUN。
- 用户需要先授权 USDT，授权完成后才能铸造 SUN。
- 单笔铸造不能超过 `10,000 USDT`。

### 2.2 卖 SUN

用户流程：

```text
SUN -> 授权 SunCurve -> 销毁 SUN -> 取回 USDT
```

前端说明：

- 用户输入想卖出的 SUN 数量。
- 前端显示预计可取回多少 USDT。
- 用户需要先授权 SUN，授权完成后才能卖出 SUN。
- SUN Burn 没有单笔上限，但不能超过用户余额。

### 2.3 铸造 MOON

用户流程：

```text
SUN -> 授权 MoonCurve -> 铸造 MOON
```

前端说明：

- 用户输入想投入的 SUN 数量。
- 前端显示预计可获得多少 MOON。
- 前端显示这笔 SUN 按当前 SUN 价格折算成多少 USDT。
- 折算后的 USDT 等值不能超过 `10,000 USDT`。
- 如果当前时间还没到 `LAUNCH_TIME`，前端应禁用铸造按钮并显示倒计时。

### 2.4 卖 MOON

用户流程：

```text
MOON -> 销毁 MOON -> 取回 SUN
```

前端说明：

- 用户输入想卖出的 MOON 数量。
- 前端显示预计可取回多少 SUN。
- MOON Burn 没有时间锁、没有单笔上限，但不能超过用户余额。
- 当前最小版合约中，卖出 MOON 不需要提前授权 MOON。

### 2.5 从 MOON 退出到 USDT

项目不提供一键 `MOON -> USDT` 功能。用户如果想从 MOON 退出到 USDT，需要分两步：

```text
MOON -> Burn 成 SUN
SUN -> Burn 成 USDT
```

前端可以用提示文案说明这条路径，但不建议做成一键操作。

## 3. 前端页面需要展示的数据

### 3.1 全局状态

| 数据 | 说明 |
| --- | --- |
| 当前连接钱包 | 用户地址 |
| 当前网络 | 本地链、测试网或其他网络 |
| Chain ID | 本地 Anvil 通常是 `31337` |
| 合约地址 | MockUSDT、SunToken、SunCurve、MoonToken、MoonCurve |
| 协议预算钱包 | 用于接收协议手续费的钱包地址 |

### 3.2 用户余额

| 数据 | 说明 |
| --- | --- |
| USDT 余额 | 用户可用于铸造 SUN 的余额 |
| SUN 余额 | 用户可用于卖出或铸造 MOON 的余额 |
| MOON 余额 | 用户可用于卖出的余额 |
| USDT 授权额度 | 是否已授权 SunCurve 使用 USDT |
| SUN 授权额度 | 是否已授权 MoonCurve 或 SunCurve 使用 SUN |
| MOON 授权额度 | 当前最小版合约不需要 MOON 授权；`MoonCurve.burn()` 会由曲线合约直接销毁用户 MOON |

### 3.3 SUN 曲线数据

| 数据 | 说明 |
| --- | --- |
| SUN 当前价格 | 按曲线账本计算，不使用 AMM 价格 |
| SUN 总供应量 | 当前 SUN 供应量 |
| SUN 储备 USDT | SUN 曲线里的 USDT 储备 |
| SUN 单笔 Mint 上限 | `10,000 USDT` |
| SUN Mint 预计获得数量 | 用户输入 USDT 后估算 |
| SUN Burn 预计取回数量 | 用户输入 SUN 后估算 |
| SUN Mint 手续费 | 总计 2% |
| SUN Burn 手续费 | 总计 2% |

### 3.4 MOON 曲线数据

| 数据 | 说明 |
| --- | --- |
| MOON 当前 Mint 价格 | 用 SUN 计价，并可折算成 USDT |
| MOON 当前 Burn 价格 | 用 SUN 计价，并可折算成 USDT |
| MOON 当前供应量 | 当前曲线状态下的 MOON 数量 |
| MOON 储备 SUN | MOON 曲线里的 SUN 储备 |
| MOON 启动时间 | `LAUNCH_TIME` |
| MOON 启动倒计时 | 未启动时显示 |
| MOON 单笔 Mint 上限 | 等值 `10,000 USDT` 的 SUN |
| MOON Mint 预计获得数量 | 用户输入 SUN 后估算 |
| MOON Burn 预计取回数量 | 用户输入 MOON 后估算 |
| MOON Mint 手续费 | 总计 5% |
| MOON Burn 手续费 | 总计 5% |

## 4. 前端页面需要提供的按钮

### 4.1 钱包和网络

- `连接钱包`
- `断开连接`
- `切换网络`
- `刷新数据`

### 4.2 本地测试专用

这些按钮只建议在 Anvil 本地链或测试环境显示：

- `领取 MockUSDT`
- `复制合约地址`
- `查看本地测试记录`

### 4.3 SUN 操作

- `授权 USDT`
- `铸造 SUN`
- `授权 SUN`
- `卖出 SUN`

### 4.4 MOON 操作

- `授权 SUN`
- `铸造 MOON`
- `卖出 MOON`

## 5. 按钮禁用规则

前端需要在以下情况禁用对应按钮：

- 钱包未连接。
- 当前网络不正确。
- 用户输入为空或为 0。
- 用户余额不足。
- 授权额度不足。
- SUN Mint 输入超过 `10,000 USDT`。
- MOON Mint 输入的 SUN 折算后超过 `10,000 USDT`。
- MOON 尚未到启动时间。
- 用户正在等待上一笔交易确认。
- 读取合约数据失败。

## 6. 页面提示文案建议

### 6.1 SUN 说明

```text
SUN 是项目的基础储备代币。你可以用 USDT 铸造 SUN，也可以销毁 SUN 换回 USDT。SUN 的曲线价格只读取合约储备账本，不使用 AMM 市场价格。
```

### 6.2 MOON 说明

```text
MOON 是项目的稀缺飞轮代币。你需要先拥有 SUN，才能用 SUN 铸造 MOON。MOON 的铸造价格会随曲线储备增加而上升。
```

### 6.3 路径说明

```text
本项目不提供 USDT 一键铸造 MOON。请先用 USDT 铸造 SUN，再用 SUN 铸造 MOON。
```

### 6.4 本地链说明

```text
当前为本地测试环境，MockUSDT 和测试账户只能用于开发测试，不能用于真实资金。
```

## 7. 暂不做的功能

第一版前端暂不建议做：

- 不做主网真实资金入口。
- 不做一键 `USDT -> MOON`。
- 不做一键 `MOON -> USDT`。
- 不做 AMM 加池入口。
- 不做收益承诺、投资建议或价格承诺。
- 不做复杂图表和交易历史。
- 不接入真实 USDT。
- 不修改 Solidity 合约代码。
- 不修改部署脚本。

## 8. 第一版页面结构建议

建议前端第一版使用一个简单页面，分成四块：

```text
顶部：钱包连接、网络状态、刷新按钮

总览：SUN 价格、MOON 价格、储备、供应量、启动倒计时

操作区：
  - 买 SUN
  - 卖 SUN
  - 铸造 MOON
  - 卖 MOON

本地测试区：
  - 领取 MockUSDT
  - 合约地址
  - 测试环境说明
```

这样做的好处是前端可以先跑通核心用户路径，再进入 Hook、AMM 和更复杂的图表阶段。

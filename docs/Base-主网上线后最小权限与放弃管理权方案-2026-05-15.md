# Base 主网上线后最小权限与放弃管理权方案 - 2026-05-15

本文档记录一个新的主网上线原则：上线完成后，项目方不保留日常管理权限。当前仍然不部署 Base 主网，不接真实资金，不收集私钥。

## 用户确认的目标

主网上线后，项目方只保留一个角色：

1. 协议经费钱包地址。

除此之外，其他管理权限全部放弃。

这里需要把“权限”和“动作”分清楚：

- 协议经费钱包地址不是管理员权限。它只有一项权益：接收协议经费。它不能改合约、不能暂停合约、不能改白名单、不能部署、不能做 owner、不能做 signer 授权其他管理动作。
- 2026-05-16 策略更新：SUN/MOON 都保持自由转账，不再试图从合约层禁止市场自行创建 AMM 池。项目只对明确支持的 Uniswap v4 Hook 池提供费用逻辑。
- 项目支持池目标改为：`SUN/USDC` v4 Hook 池 swap 收 `2% USDC`，`MOON/USDC` v4 Hook 池 swap 收 `5% USDC`。第三方自行创建的池不代表协议价格，也不保证触发项目费用和 SUN 曲线回灌。
- 任何能改参数、改白名单、改 adapter、暂停合约、转移 owner、重新设置收款地址的能力，都属于需要放弃的管理权限。

## 一句话结论

如果主网坚持这个最小权限目标，当前已部署到 Base Sepolia 的旧版 `BaseMoonAmmFeeV4Hook` 不能直接作为主网版本。原因是旧 Hook 的 owner 是 immutable 地址，没有 `transferOwnership()` 或 `renounceOwnership()`，上线后无法通过一笔交易放弃 Hook 管理权限；它也只覆盖旧 `MOON` 费用方向。2026-05-16 已新增本地版 `BaseSunMoonUsdcFeeV4Hook`，用于统一支持 `SUN/USDC` 2% 和 `MOON/USDC` 5% 两类 v4 Hook 池，并支持 renounce；本地 poolId 计算脚本也已完成，但主网前仍需用正式参数重新挖 salt、生成正式 poolId、做 fork dry-run 和外部 review。

因此，主网前必须新增一个 Gate：

```text
Gate D = 主网无管理员权限方案完成
```

Gate D 未完成前，不进入主网广播。

## 最终主网权限目标

主网上线完成后，理想链上状态应该是：

| 对象 | 上线后状态 | 说明 |
| --- | --- | --- |
| `PROTOCOL_BUDGET_WALLET` | 保留 | 唯一权益是接收协议经费；不拥有任何管理权 |
| SUN/MOON 自由转账 | 保留 | 不从 token 合约层禁止市场自行建池 |
| `SUN/USDC` v4 Hook 池 | 支持 | 项目支持路径，swap 收 `2% USDC` |
| `MOON/USDC` v4 Hook 池 | 支持 | 项目支持路径，swap 收 `5% USDC` |
| `SunToken.owner()` | `address(0)` 或不可管理状态 | `minter` 已锁定后不再需要 owner |
| `MoonToken.owner()` | `address(0)` 或不可管理状态 | `minter` 已锁定后不再需要 owner |
| `SunCurve.owner()` | `address(0)` 或不可管理状态 | `moonCurve`、`moonAMM` 绑定后不再需要 owner |
| `MoonCurve.owner()` | `address(0)` 或不可管理状态 | 当前没有上线后必须保留的 owner 操作 |
| `DirectUsdcOnlyAdapter.owner()` | `address(0)` 或不可管理状态 | `authorizedHook` 固定后不再允许改 |
| `BaseMoonAmmFeeV4Hook.owner()` | `address(0)` 或不可管理状态 | `protocolBudget`、`swapAdapter`、`MOON/USDC poolId` 固定后不再允许改 |
| `SunAmmGuardHook` / `BaseSunAmmGuardV4Hook` | 不进入主网部署 | 作为历史测试原型保留，不作为主网组件 |
| `Create2HookDeployer.owner` | 不作为运行期权限 | 部署工具不应控制已上线协议；如保留也不能持有资金或被当作管理员 |

## 当前代码里的冲突点

### 可以通过 Ownable 放弃的合约

这些合约继承 OpenZeppelin `Ownable`，理论上可以在配置完成后调用 `renounceOwnership()`：

- `SunToken`
- `MoonToken`
- `SunCurve`
- `MoonCurve`
- `DirectUsdcOnlyAdapter`
- `TestnetUsdcAdapter`，仅测试网使用

放弃前必须先完成必要配置：

- `SunToken.setMinter(SunCurve)`
- `MoonToken.setMinter(MoonCurve)`
- `SunCurve.setMoonCurve(MoonCurve)`
- `SunCurve.setMoonAMM(Hook)`
- `DirectUsdcOnlyAdapter.setAuthorizedHook(Hook)`

`SunAmmGuardHook` / `BaseSunAmmGuardV4Hook` 作为历史“首次加池守卫”原型保留。2026-05-16 新方向不再依赖“禁止加池”模型，主网前应改为统一 USDC 费用 Hook 设计。

### 当前不能直接放弃的合约

`BaseMoonAmmFeeV4Hook` 当前使用：

```solidity
address public immutable owner;
```

并且有这些 owner 权限：

- `setProtocolBudget()`
- `setSwapAdapter()`
- `setAllowedMoonPool()`
- `setPaused()`

因为 owner 是 immutable，所以部署后不能转移，也不能 renounce。只要 owner 地址仍然可用，就仍然有人能改 Hook 配置。这和“其他全部放弃管理权限”的主网目标冲突。

## 主网前推荐改动

主网前推荐把 `BaseMoonAmmFeeV4Hook` 做一次最小治理改造：

```text
BaseMoonAmmFeeV4Hook v3:
  - 支持 owner()
  - 支持 transferOwnership(newOwner)
  - 支持 renounceOwnership()
  - 保持 onlyOwner 保护 setProtocolBudget / setSwapAdapter / setAllowedMoonPool / setPaused
  - renounce 后 owner == address(0)
  - renounce 后所有 onlyOwner 配置函数永久不可调用
```

对应测试必须覆盖：

- 配置完成前 owner 可以设置 `protocolBudget`、`swapAdapter`、`allowedMoonPool`、`paused`。
- `renounceOwnership()` 后 owner 变成 `address(0)`。
- `renounceOwnership()` 后不能再改 `protocolBudget`。
- `renounceOwnership()` 后不能再改 `swapAdapter`。
- `renounceOwnership()` 后不能再改 `allowedMoonPool`。
- `renounceOwnership()` 后不能再暂停或恢复。
- 已白名单的 `MOON/USDC` 池在 renounce 后仍然可以正常执行 swap fee 路由。
- 未白名单的 `MOON/*` 池在 renounce 后仍然不能交易。

注意：Hook 字节码改变后，CREATE2 预测地址会改变，必须重新挖 salt，重新做权限 bit 检查，重新跑 Base Sepolia 或 fork 演练。主网前这样改是可以的；主网上线后再改就会很痛苦。

## 协议经费钱包的边界

`PROTOCOL_BUDGET_WALLET` 只能做收款地址，不能同时做管理员地址。

硬性规则：

```text
PROTOCOL_BUDGET_WALLET 权益 = 接收协议经费
PROTOCOL_BUDGET_WALLET 权限 = 无
```

它不应该出现在任何 owner、deployer、pause、adapter admin、CREATE2 owner、前端管理员或白名单管理字段里。

推荐边界：

```text
PROTOCOL_BUDGET_WALLET:
  - 接收 SUN mint/burn 协议费
  - 接收 MOON mint/burn 协议费
  - 接收 SUN/USDC v4 Hook 池协议费
  - 接收 MOON/USDC v4 Hook 池协议费
  - 不做 owner
  - 不做 deployer
  - 不做 adapter owner
  - 不做 pause guardian
  - 不做 MAINNET_ADMIN_WALLET 的同一个普通钱包
  - 不参与任何合约配置投票
```

如果 Hook 当前还允许 `setProtocolBudget()`，则必须在最终地址确认后 renounce Hook owner。否则协议经费地址仍然可能被管理员改掉。

## SUN/MOON 自由转账与 v4 Hook 池收费

2026-05-16 确认的新规则：

```text
SUN_TRANSFER_POLICY=free-transfer
MOON_TRANSFER_POLICY=free-transfer
MARKET_AMM_CREATION=not-controlled-by-protocol
PROJECT_SUPPORTED_POOLS=SUN/USDC v4 Hook pool + MOON/USDC v4 Hook pool
```

这意味着：

- SUN 和 MOON 都保持 ERC20 自由转账。
- 市场可以自行创建 SUN 或 MOON 的 AMM 池，协议不承诺也不试图完全阻止。
- 项目不把第三方池价格当成协议价格。
- SUN 的协议价格仍然来自 `SunCurve.curveReserve / SunToken.totalSupply`。
- 项目支持路径只指明确配置的 v4 Hook 池。

项目支持池的建议费用：

```text
SUN/USDC v4 Hook 池:
  swap fee = 2% USDC
  1.5% USDC -> SunCurve.injectUSDT()
  0.5% USDC -> PROTOCOL_BUDGET_WALLET

MOON/USDC v4 Hook 池:
  swap fee = 5% USDC
  3% USDC -> SunCurve.injectUSDT()
  2% USDC -> PROTOCOL_BUDGET_WALLET
```

技术建议：

```text
BaseSunMoonUsdcFeeV4Hook:
  - 支持 owner() / transferOwnership() / renounceOwnership()
  - 支持配置 SUN/USDC poolId
  - 支持配置 MOON/USDC poolId
  - 只对 USDC 计费路径负责
  - 配置完成后 owner 放弃，后续不能新增或修改支持池
```

如果继续沿用 `SunCurve.moonAMM` 字段，文档中必须说明它实际代表“授权 USDC 注入 Hook”。后续代码更理想的命名是 `feeInjector` 或 `authorizedAmmHook`。

## 主网执行顺序草案

不涉及私钥，只描述公开流程：

1. 确认 SUN/MOON 都保持自由转账。
2. 确认主网第一阶段项目支持池为 `SUN/USDC` 和 `MOON/USDC` 两个 v4 Hook 池。
3. 完成统一 `BaseSunMoonUsdcFeeV4Hook` 或等效 Hook 方案。
4. 完成 Hook 放弃 owner 改造。
5. 跑全量测试，新增 SUN/USDC 2%、MOON/USDC 5%、renounce 后行为测试。
6. 重新挖 CREATE2 salt，确认 Hook 权限 bit。
7. 用 Base mainnet fork 做完整 dry-run，不广播。
8. 确认正式 `PROTOCOL_BUDGET_WALLET` 公开地址。
9. 部署合约并完成所有必要绑定。
10. 配置正式 `SUN/USDC` 和 `MOON/USDC` poolId。
11. 逐个合约执行 `renounceOwnership()` 或等效放弃管理权动作。
12. 链上复核所有 owner 状态。
13. owner 放弃复核通过后，才允许公开主网交易入口。

# 安全检查清单

## 1. 总原则

智能合约一旦部署到链上，很多错误无法像普通网站一样轻松修复。项目必须先在本地和测试网充分验证，再考虑主网部署。

本清单用于每次开发、测试、部署前逐项检查。没有完成检查前，不建议部署到真实资金环境。

## 2. 密钥安全

- [ ] 私钥没有写入代码。
- [ ] 私钥没有写入文档。
- [ ] 私钥没有出现在截图或聊天记录中。
- [ ] `.env` 文件已经加入 `.gitignore`。
- [ ] 部署钱包只保留必要测试资金。
- [ ] 主网部署钱包和日常测试钱包分开。
- [ ] 不使用来历不明的钱包插件或签名工具。

## 3. 依赖安全

- [ ] 只使用可信来源的合约依赖。
- [ ] 记录 Uniswap v4 core 版本。
- [ ] 记录 Uniswap v4 periphery 版本。
- [ ] 记录 OpenZeppelin Contracts 版本。
- [ ] 不复制粘贴不明来源的合约代码。
- [ ] 更新依赖后重新运行完整测试。

## 4. 权限控制

- [ ] 管理员函数只允许管理员调用。
- [ ] 非管理员调用管理员函数会失败。
- [ ] 管理员地址初始化正确。
- [ ] 如果支持转移管理员，已测试转移流程。
- [ ] 如果支持放弃管理员，已确认不会导致项目无法维护。
- [ ] 高风险操作有事件记录。

## 5. 暂停和应急机制

- [ ] 合约支持暂停或紧急关闭核心功能。
- [ ] 暂停后高风险操作会被阻止。
- [ ] 恢复后正常功能可继续使用。
- [ ] 前端能显示当前暂停状态。
- [ ] 应急操作流程写入文档。

## 6. 参数安全

- [ ] 所有配置参数都有合理范围。
- [ ] 数值参数不能设置为明显危险值。
- [ ] 地址参数不能误设为零地址。
- [ ] 修改参数会发出事件。
- [ ] 前端对参数做基础校验。
- [ ] 测试覆盖边界值。

## 7. Hook 权限和地址

- [ ] Hook 启用的回调函数与合约地址权限匹配。
- [ ] 部署脚本能生成或验证正确 Hook 地址。
- [ ] 测试中检查 Hook 权限配置。
- [ ] 只对预期池子启用 Hook。
- [ ] 不允许未知池子绕过关键限制。

## 8. 交易逻辑安全

- [ ] 正常交易路径测试通过。
- [ ] 异常交易路径测试通过。
- [ ] 暂停状态下交易行为符合预期。
- [ ] 超过限制的交易会被正确处理。
- [ ] 不会因为舍入误差造成明显漏洞。
- [ ] 不依赖容易被操纵的单一区块价格。
- [ ] 不在 Hook 中执行不必要的外部调用。

## 9. 重入风险

- [ ] 检查是否存在外部合约调用。
- [ ] 外部调用前后状态更新顺序合理。
- [x] 必要时使用重入保护。
- [x] 测试恶意合约回调场景。

## 10. 资金安全

- [ ] 合约是否会持有用户资金已明确。
- [ ] 如果合约持有资金，提款权限和流程已测试。
- [ ] 任何费用收取逻辑都有清楚记录。
- [ ] 不存在管理员随意提走用户资金的隐藏路径。
- [ ] 不承诺固定收益。

## 11. 前端安全

- [ ] 前端显示当前网络。
- [ ] 前端在错误网络禁止高风险操作。
- [ ] 前端显示交易确认状态。
- [ ] 前端不保存私钥。
- [ ] 前端不诱导用户签署不明交易。
- [ ] 合约地址从可信配置读取。

- [ ] 合约已完整编译。
- [ ] 测试全部通过。
- [ ] 部署网络确认无误。
- [ ] 部署钱包确认无误。

- [ ] 部署记录已保存。
- [ ] 区块浏览器验证计划已准备。

## 14. 主网上线前检查

- [ ] 至少完成一次测试网完整演练。
- [ ] 完成安全检查清单。
- [ ] 关键合约经过外部 review 或审计。
- [ ] 前端连接的是正确主网合约地址。
- [ ] 用户风险提示清晰。
- [ ] 准备应急联系人和应急流程。
- [ ] 明确是否可以升级，若可升级，需要说明升级权限和风险。

## 15. 每次修改后的回归检查

- [ ] 修改点对应测试已新增或更新。
- [ ] 全部测试重新运行。
- [ ] 文档同步更新。
- [ ] 前端配置同步更新。
- [ ] 没有引入新的私钥、临时地址或测试数据。

13. 初始化 `SUN/USDC` v4 池。

16. 只读确认 Hook 余额为 0 或符合预期。

18. 只读确认 `Hook.owner() == address(0)`。

## 11. 权限锁定的明确决定

我的明确决定：

- [ ] Hook 必须最终 renounce，不能长期保留 owner。
- [ ] Hook renounce 必须放在所有 allowlist、pool 初始化、`SunCurve.moonAMM` 绑定、链上只读复核之后。
- [ ] `SunToken.minter` 和 `MoonToken.minter` 必须在部署阶段锁定，且已经由测试覆盖。
- [ ] `PROTOCOL_BUDGET_WALLET` 只收款，不做 owner。
- [ ] `SunCurve` / `MoonCurve` / Token owner 的进一步 renounce 不和第一批部署交易混在一起；需要先单独写 renounce-only 预检清单和测试，再做最终锁定。

原因：

```text
Hook owner 能改 allowlist、protocolBudget、pause，是上线后最敏感权限。
所以 Hook 必须在确认无误后锁掉。

Token minter 一旦 setMinter 会永久锁定，这已经满足核心铸币权限锁定。

SunCurve/MoonCurve/Token 的 owner renounce 也可以作为最终去中心化目标，
但它是不可逆动作，不应该混在首次主网部署批次里临时操作。
```
